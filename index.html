.Git Ignore

Es un archivo de texto utilizado en proyectos de Git para especificar qué archivos o directorios deben ser ignorados por el control de versiones. Esto significa que Git no rastreará ni incluirá estos archivos en los commits, incluso si cambian.

Pasos para utilizar

  1. Crear el archivo .gitignore: En la raíz de tu proyecto, crea un archivo llamado .gitignore.
  2. Añadir reglas al .gitignore: Abre el archivo y escribe las reglas para ignorar archivos o carpetas (por ejemplo, *.log para ignorar archivos .log).
  3. Guardar y cerrar el archivo: Guarda los cambios en el archivo .gitignore.
  4. Eliminar archivos previamente añadidos (si es necesario): Si ya hay archivos rastreados que deberían ser ignorados, elimina su seguimiento con git rm --cached archivo.
  5. Hacer commit de los cambios: Añade el archivo .gitignore a Git y haz un commit: 
    git add .gitignore
    git commit -m "Añadir .gitignore"
  6. Continuar trabajando normalmente: Git ignorará los archivos especificados en .gitignore en futuros commits.
